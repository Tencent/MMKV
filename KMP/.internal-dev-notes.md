# KMP internal development notes

This file is for maintainers/debugging only. Keep user-facing setup and release information in `README.md` and `CHANGELOG.md`.

## Darwin dependency path

KMP Darwin no longer uses CocoaPods. iOS/macOS targets use the same `Core/cbridge/MMKVBridge.h` cinterop path as other Kotlin/Native targets.

Relevant files:
- `KMP/mmkv/build.gradle.kts` wires Darwin cinterop and CMake builds.
- `KMP/mmkv/nativeInterop/cinterop/mmkv.def` declares the C bridge header/static library and Darwin linker options.
- `KMP/mmkv/src/darwinMain/kotlin/com/tencent/mmkv/kmp/MMKV.darwin.kt` wraps the C bridge.
- `KMP/mmkv/src/darwinMain/kotlin/com/tencent/mmkv/kmp/MMKVNameSpace.darwin.kt` wraps namespace APIs through the C bridge.

## Dependency source overrides

Inside the monorepo, CMake uses local MMKV source. Outside the monorepo, CMake can fetch via:
- `MMKV_GIT_REPOSITORY`
- `MMKV_GIT_TAG`
- `MMKV_GIT_BRANCH`
- `MMKV_GIT_COMMIT`

Do not reintroduce `MMKV_POD_SOURCE`; it was only for the old CocoaPods cinterop path.

## Darwin lifecycle debugging

The old ObjC/CocoaPods wrapper path could crash after `MMKV.close()` plus `removeStorage()` because Kotlin/Native GC later released an ObjC `MMKV` wrapper whose native pointer had been invalidated elsewhere. Moving Darwin KMP to the C bridge avoids the ObjC wrapper lifetime path.

If a similar crash appears, inspect whether a Kotlin wrapper still references a native handle after `mmkv_close()` or `removeStorage()` has destroyed the underlying instance.

## JVM desktop ABI reminder

JNA `Boolean` does not match C `bool` ABI. For C bridge calls from JVM desktop:
- use `Byte` for C `bool` parameters,
- use `Byte` for C `bool` fields in `Structure`,
- convert Kotlin booleans with `toNativeByte()`.

This matters for encryption paths such as `mmkv_rekey()` and `MMKVConfig_t.aes256`.

## Verification commands

Useful focused checks:

```bash
cd KMP
./gradlew :mmkv:compileKotlinMacosX64 --stacktrace
./gradlew :mmkv:compileKotlinIosArm64 --stacktrace
./gradlew :mmkv:compileKotlinIosSimulatorArm64 --stacktrace
./gradlew :mmkv:macosX64Test --stacktrace
./gradlew :mmkv:desktopTest --stacktrace
./gradlew :sample:composeApp:linkDebugFrameworkIosArm64 --stacktrace
xcodebuild -project sample/iosApp/iosApp.xcodeproj -scheme iosApp -configuration Debug -destination 'generic/platform=iOS' CODE_SIGNING_ALLOWED=NO build
```
